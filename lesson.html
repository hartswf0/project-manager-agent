Project Management Assistant: Learnings and Code Breakdown

Our team recently completed the development of a Project Management Assistant chatbot. Here's what we learned during the process, with key code snippets and explanations.

Frontend Developer's Insights

As the frontend developer, I focused on creating an intuitive and responsive user interface. Here are some key learnings:

1. Responsive Design

We used flexbox to create a responsive layout that adapts to different screen sizes:


body {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}
.container {
    max-width: 1000px;
    margin: auto;
    flex-grow: 1;
}

This ensures that our app looks good on both desktop and mobile devices.

2. Dynamic UI Updates

We implemented dynamic UI updates for project phases:


function setActivePhase(phase) {
    currentPhase = phase;
    document.querySelectorAll('.phase').forEach(el => el.classList.remove('active'));
    document.querySelector(`[data-phase="${phase}"]`).classList.add('active');
    addMessageToChat('System', `Switched to ${phase} phase. How can I assist you with this phase of your project?`);
}

This function updates the UI and provides immediate feedback when a user switches project phases.

Lesson Learned:

Combining CSS transitions with JavaScript for UI updates creates a smooth and engaging user experience. Always consider the user's perspective when designing interactions.

Backend Developer / API Specialist's Insights

My role involved integrating the OpenAI API and managing the communication between the frontend and the AI service. Here are some key learnings:

1. API Key Security

We implemented client-side API key validation to improve user experience:


async function testApiKey() {
    const apiKey = apiKeyInput.value.trim();
    if (!apiKey) {
        log('Please enter an API key.');
        return;
    }

    log('Testing API key...');
    try {
        const response = await fetch('https://api.openai.com/v1/engines', {
            headers: {
                'Authorization': `Bearer ${apiKey}`
            }
        });

        if (response.ok) {
            log('API key is valid!');
        } else {
            log('API key is invalid. Please check and try again.');
        }
    } catch (error) {
        log(`Error testing API key: ${error.message}`);
    }
}

This function checks if the API key is valid before allowing the user to send messages.

2. Error Handling in API Requests

We implemented robust error handling for API requests:


try {
    const response = await fetch('https://api.openai.com/v1/chat/completions', {
        // ... request details ...
    });

    if (!response.ok) {
        throw new Error(`HTTP error! status: ${response.status}`);
    }

    // ... process response ...
} catch (error) {
    log(`Error: ${error.message}`);
    addMessageToChat('Assistant', 'Sorry, there was an error processing your request.');
}

This ensures that users receive meaningful feedback if something goes wrong with the API request.

Lesson Learned:

Always implement thorough error handling when working with external APIs. It significantly improves the reliability and user experience of your application.

AI/NLP Engineer's Insights

My focus was on optimizing the AI responses and ensuring the chatbot provided relevant advice for each project phase. Here's what I learned:

1. Dynamic System Prompts

We implemented dynamic system prompts based on the current project phase:


function getSystemPrompt() {
    const basePrompt = `You are an AI assistant specialized in guiding teams through the process of developing technical projects. ...`;
    
    let phaseSpecificPrompt = '';
    switch(currentPhase) {
        case 'brainstorm':
            phaseSpecificPrompt = `Focus on helping the team define clear objectives, identify target audience and use cases, and prioritize features. ...`;
            break;
        // ... other phases ...
    }

    return basePrompt + ' ' + phaseSpecificPrompt;
}

This function generates a tailored system prompt for each project phase, improving the relevance of AI responses.

Lesson Learned:

Customizing AI prompts based on context significantly enhances the quality and relevance of AI-generated responses. It's crucial to carefully craft these prompts to guide the AI effectively.

<div id="console-output"></div>
        

function log(message) {
